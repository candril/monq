# Bulk Edit and Insert

**Status**: Done

## Description

Edit multiple selected documents simultaneously in `$EDITOR` as a JSON array. Detects documents removed from the array (delete from DB) or added to the array (insert into DB) and presents a confirmation dialog before applying side effects. Also supports inserting a new document from a blank template derived from the collection schema.

## Out of Scope

- Inline multi-field editing without leaving the TUI
- Partial field updates across selection (patch mode)

## Capabilities

### P1 - Must Have

- `e` with active selection opens all selected documents in `$EDITOR` as a JSON array
- Detects removed entries (deleted from array) → prompts to delete from DB
- Detects added entries (new objects added to array) → prompts to insert into DB
- Confirmation dialog with choices: **back** / **skip side effects** / **delete** / **insert** / **both** / **cancel**
- Reload documents after applying edits

### P2 - Should Have

- JSON Schema sidecar `.monq-docs-schema.json` generated alongside the temp file — provides per-field type completions for the editor's LSP (jsonls, vim-lsp, etc.) — **done**
- Schema includes all known field paths with detected types from the sampled collection schema

### P3 - Nice to Have

- Diff summary before confirming (N modified, N deleted, N added)

## Implementation Notes

### Temp Files

- Edit temp: `$TMPDIR/monq/{collection}/edit-{timestamp}.json` — JSON array of selected docs
- Schema sidecar: `$TMPDIR/monq/{collection}/.monq-docs-schema.json` — JSON Schema for the array items

### Insert Flow

- `i` key opens editor with a blank template object derived from the current collection's `schemaMap`
- Template fills string fields with `""`, number fields with `0`, boolean fields with `false`, etc.
- On save+quit: calls `insertDocument()` for each new object in the array
- Reloads documents after insert

### Schema Sidecar

Generated by `openEditorForMany()` / `openEditorForInsert()` in `src/actions/editMany.ts`. Mirrors the pipeline schema sidecar pattern from spec 012 but scoped to document fields rather than pipeline stages.

### Confirm Dialog

`ConfirmChoiceDialog` variant of `ConfirmDialog` presents labeled key-press choices:
- `b` → back (re-open editor)
- `s` → skip side effects (save modifications only, ignore removed/added)
- `d` → delete removed docs
- `i` → insert added docs
- `a` → both (delete + insert)
- `Escape` → cancel (no changes applied)

## Key Files

- `src/actions/editMany.ts` — `openEditorForMany()`, `openEditorForInsert()`, `applyConfirmActions()`
- `src/hooks/useKeyboardNav.ts` — `e` (selection-aware), `i` key handlers; bulk-edit confirm dialog handler
- `src/components/ConfirmDialog.tsx` — `ConfirmChoiceDialog` component
- `src/providers/mongodb.ts` — `insertDocument()`, `deleteDocument()`, `replaceDocument()`
- `src/query/schema.ts` — `buildSchemaMap()` used to generate template and LSP schema

## Keyboard

| Key | Context | Action |
|-----|---------|--------|
| `e` | Selection active | Open all selected docs in $EDITOR |
| `e` | No selection | Open current doc in $EDITOR (single edit, spec 007) |
| `i` | Document list | Open editor with blank insert template |
